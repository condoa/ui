# FormSelect

A Select wired to VeeValidate, with a label, optional icon, and a hint or error message below it. Use inside forms when you want validation state handled for you.

## When to Use
- A select field inside a VeeValidate form
- You want label, hint, and error message in one component
- Validation state should drive the field styling
- NOT for a bare select with no validation (use Select or NativeSelect)
- NOT for searchable lists (use Combobox)

## Usage

Basic:
```vue
<FormSelect v-model="status" name="status" label="Status" :rules="required">
    <SelectContent>
        <SelectItem value="open">Open</SelectItem>
        <SelectItem value="closed">Closed</SelectItem>
    </SelectContent>
</FormSelect>
```

The default slot replaces the inner trigger and content, so pass `SelectContent`
(and optionally a custom `SelectTrigger`) as children. A default trigger with the
placeholder is rendered when you do not.

## Props
- `v-model` (`modelValue`) - selected value
- `name` - field name, also the VeeValidate field key
- `label` - label text shown above the select
- `icon` - icon name shown inside the field
- `hint` - helper text shown below when there is no error
- `id` - id of the select (auto-generated when omitted)
- `rules` - VeeValidate rules for the field
- `validateOnMount` - validate immediately on mount
- `placeholder` - placeholder for the default trigger
- `disabled` - disable the select
- `required` - mark the field as required

Unknown attributes are forwarded to the inner Select (inheritAttrs is off).

## Slots
- default - replaces the inner trigger and content (pass SelectContent here)
- `trigger` - custom trigger element
- `content` - custom content, used when the default slot is not provided
- `icon` - replace the leading icon area
- `hint` - scoped slot, receives `{ errorMessage, value }`
- `errorMessage` - scoped slot, receives `{ errorMessage, value }`

## Validation
- Uses VeeValidate `useField` keyed by `name` (or the generated id)
- Reads `rules` and `validateOnMount`
- The label turns destructive when there is an error
- The hint is hidden while an error message is shown

## Accessibility
- The label is associated with the select via `for`/`id`
- Error and hint text animate in below the field
- Built on the Select primitive, so ARIA combobox semantics apply

## Common Patterns

With hint and icon:
```vue
<FormSelect
    v-model="role"
    name="role"
    label="Role"
    icon="icon-user"
    hint="Pick the access level"
    :rules="required"
>
    <SelectContent>
        <SelectItem value="admin">Admin</SelectItem>
        <SelectItem value="member">Member</SelectItem>
    </SelectContent>
</FormSelect>
```

## Don't
- Don't use outside a VeeValidate form (there is no field context)
- Don't pass options without SelectContent in the default slot
- Don't set both a custom `trigger` slot and rely on `placeholder`

## Related
