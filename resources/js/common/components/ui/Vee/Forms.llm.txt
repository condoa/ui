# VeeValidate Form Components

Form inputs integrated with VeeValidate for client-side validation. All components auto-register with the parent useForm() context and display validation errors.

## When to Use
- Forms requiring client-side validation
- Inertia forms with VeeValidate integration
- NOT for forms with server-only validation (use plain Input, Textarea, etc.)

## Critical: One useForm() Per Component

VeeValidate uses provide/inject. If you call useForm() multiple times in one component, only the LAST form context is active. All VeeInput/VeeSelect fields register with that last form.

Wrong:
```vue
<script setup>
const form1 = useForm({ schema: schema1 }) // Lost!
const form2 = useForm({ schema: schema2 }) // All fields use this
</script>
```

Right (separate components or drop VeeValidate):
```vue
<!-- Option A: Extract forms to child components -->
<Form1Dialog />
<Form2Dialog />

<!-- Option B: Use plain inputs with Inertia errors -->
<Input v-model="form.name" :error="form.errors.name" />
```

## Usage with Inertia

Dual form pattern (VeeValidate + Inertia):
```vue
<script setup>
import { useForm as useVeeForm } from 'vee-validate'
import { useForm as useInertiaForm } from '@inertiajs/vue3'
import * as yup from 'yup'

const schema = yup.object({
    name: yup.string().required().label('Name'),
    email: yup.string().email().required().label('Email'),
})

const { handleSubmit, setFieldValue } = useVeeForm({ validationSchema: schema })
const form = useInertiaForm({ name: '', email: '' })

const submit = handleSubmit((values) => {
    form.post(route('users.store'))
})
</script>

<template>
    <form @submit.prevent="submit">
        <VeeInput name="name" label="Name" v-model="form.name" required />
        <VeeInput name="email" label="Email" v-model="form.email" type="email" required />
        <Button :loading="form.processing">Save</Button>
    </form>
</template>
```

## Components

### VeeInput
Text input with label, validation, and icons.
```vue
<VeeInput
    name="email"
    label="Email"
    type="email"
    placeholder="Enter email"
    hint="We'll never share your email"
    icon="icon-mail"
    required
/>
```

Props:
- `name`: field name (required for validation)
- `label`: label text
- `labelHint`: hint next to label
- `type`: input type (default: "text")
- `placeholder`: placeholder text
- `hint`: helper text below input
- `icon`: leading icon
- `trailingIcon`: trailing icon
- `disabled`: disable input
- `required`: show required indicator
- `rules`: validation rules (Yup or VeeValidate)
- `validateOnMount`: validate immediately

### VeeTextarea
Multi-line text input.
```vue
<VeeTextarea
    name="description"
    label="Description"
    :rows="4"
    placeholder="Enter description..."
/>
```

Props: Same as VeeInput plus:
- `rows`: number of rows

### VeeSelect
Native HTML select with validation.
```vue
<VeeSelect name="country" label="Country" required>
    <option value="">Select country</option>
    <option value="BR">Brazil</option>
    <option value="US">United States</option>
</VeeSelect>
```

Props: Same as VeeInput plus:
- `trailingIcon`: custom dropdown icon

Note: Uses native `<select>`. For styled dropdowns, use the reka-ui Select components without VeeValidate and rely on server validation.

## Accessibility

- All inputs have associated labels
- Error messages linked via aria-invalid and aria-describedby, and marked with role="alert"
- Required fields marked with asterisk and aria-required
- Focus management on validation errors
- Disabled states properly conveyed

## Common Patterns

Conditional validation:
```vue
const schema = yup.object({
    hasDiscount: yup.boolean(),
    discountCode: yup.string().when('hasDiscount', {
        is: true,
        then: (schema) => schema.required(),
    }),
})
```

Async validation:
```vue
const schema = yup.object({
    email: yup.string().email().test(
        'unique',
        'Email already in use',
        async (value) => {
            const { data } = await axios.get(`/api/check-email?email=${value}`)
            return data.available
        }
    ),
})
```

Setting values programmatically:
```vue
// IMPORTANT: Must use setFieldValue alongside form assignment
const { setFieldValue } = useVeeForm({ validationSchema: schema })

function selectUnit(unit) {
    form.unit_id = unit.id
    setFieldValue('unit_id', unit.id) // Required for VeeValidate to know!
}
```

## Don't

- Don't use multiple useForm() in one component
- Don't forget setFieldValue when setting values programmatically
- Don't mix Vee and non-Vee inputs in same validated form
- Don't skip the name prop (validation won't work)
- Don't use VeeSelect with :options prop, it uses slots
