# Vee MarkdownEditor

VeeValidate-bound editor that gives a rich-text editing experience but stores Markdown. Tiptap drives the contenteditable area; `marked` converts the stored Markdown to HTML on load and `turndown` converts edits back to Markdown. Registers a field via `useField`, so it joins the surrounding VeeValidate form. Outputs Markdown through both `v-model` and the field value. Depends on the Label and Form registry items, plus `@tiptap/vue-3`, `@tiptap/starter-kit`, the underline and placeholder Tiptap extensions, `marked`, and `turndown`.

## When to Use
- A form field whose stored value must be Markdown, edited with a WYSIWYG toolbar
- Content rendered later as Markdown (docs, notes, templates)
- NOT when you need HTML storage (use Vee HtmlEditor)
- NOT for plain text (use Vee Textarea)

## Usage
```vue
<MarkdownEditor
    name="notes"
    :label="__('templates.body')"
    :rules="bodyRules"
    v-model="form.notes"
    :hint="__('templates.body_hint')"
    placeholder="Escreva em Markdown..."
    required
/>
```

## Props
- `name`: VeeValidate field name (falls back to a generated id)
- `id`: explicit field id (otherwise generated with `useId`)
- `label`: label text shown above the editor
- `modelValue`: bound Markdown string
- `rules`: VeeValidate rules for the field
- `hint`: helper text shown below when there is no error
- `placeholder`: empty-state placeholder inside the editor
- `required`: shows a `*` next to the label

## Events
- `update:modelValue`: emits the current Markdown on every edit

## Slots
- `hint`: replaces the default hint line; receives `{ errorMessage, value }`
- `errorMessage`: replaces the default error line; receives `{ errorMessage, value }`

## Behavior
- The toolbar toggles bold, italic, underline, H2, bullet and ordered lists, blockquote, and horizontal rule; there is no link button (unlike Vee HtmlEditor)
- On load and on external `modelValue` changes, Markdown is parsed to HTML for editing; edits are serialized back to Markdown (ATX headings, `-` bullet marker)
- External changes are applied only when the serialized Markdown differs from the current content, avoiding cursor resets
- The wrapper border turns destructive on error and shows a focus ring while the editor is focused

## Don't
- Don't expect SSR; Tiptap mounts a ProseMirror editor in the browser only
- Don't round-trip exotic HTML and expect lossless Markdown; Turndown maps only the supported marks
- Don't place two `useField` wrappers under one form expecting both to bind; VeeValidate provides a single form context

## Related
- Vee HtmlEditor: same toolbar plus a link button, stores HTML instead of Markdown
- Label: the field label
- Form: provides the shared hint and error styling
